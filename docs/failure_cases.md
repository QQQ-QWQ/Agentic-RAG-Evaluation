# 典型失败案例

## Q005：改写后把 Self-RAG 答偏成了 CRAG

期望：

- 应该回答 `Self-RAG`
- 重点应放在自反思、自我判断、按需检索这些机制

实际：

- `c0_naive` 基本答对
- `c1_rewrite` 改写后答成了 `CRAG`

说明：

- 这是很典型的“改写后概念漂移”
- 说明查询改写不一定带来增益，也可能把问题带偏

## Q018：代码题两边都失败

期望：

- 应该读出 `agentic_rag_knowledge_crawler.py` 里 `save_md(...)` 的作用
- 还要说明保存时记录了哪些元信息

实际：

- `C0` 和 `C1` 都没有把关键代码片段找出来

说明：

- 这是本轮最明显的代码检索/代码理解失败案例
- 说明当前链路对代码题支持还不稳

## Q013：论文分类统计题没做出来

期望：

- 先按论文列表做人为归类
- 再统计不同类型数量

实际：

- `C0` 和 `C1` 都没有完成

说明：

- 这类题不只是检索问题
- 还需要进一步整理、归类和汇总
- 可以看作当前能力边界

## Q007：找到一些项目，但推荐不完整

期望：

- 给出更完整的开源项目推荐

实际：

- `c0_naive` 只答到 `RAGFlow`
- `c1_rewrite` 提到了 `A-RAG` 和 `RAGFlow`，但还是不完整

说明：

- 这类题的难点在于多文档整理
- 不是只要命中文档就一定能答好
